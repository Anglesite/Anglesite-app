# Apple Help (Anglesite Help Book) — Design

**Date:** 2026-05-28
**Status:** Approved
**Phase:** 10 — v2 polish (Settings polish / App Store readiness)

## Goal

Ship a real macOS **Help Book** for Anglesite: the HTML bundle that powers
**Help → Anglesite Help**, with light/dark Apple-native styling and a `hiutil`-built
search index so the Help-menu search field works. Full task-based coverage of every
shipped feature, authored as hand-written HTML.

Format decision: **classic indexed Apple Help Book** (over a plain HTML folder, or a
Markdown→HTML toolchain). It is the only option that delivers true Apple Help —
Help-menu integration, Help Viewer, and searchable help — and the indexing step is a
small script using `/usr/bin/hiutil`, already on the build machine. A Markdown
toolchain was rejected to keep the repo's tooling lean (CLAUDE.md: "no frameworks
beyond Apple's").

## Bundle structure

A standard indexed Help Book at `Resources/Anglesite.help/`:

```
Anglesite.help/
└── Contents/
    ├── Info.plist                      # help-book manifest (HPDBook* keys)
    └── Resources/
        ├── AnglesiteHelp.png           # book icon shown in Help Viewer (placeholder)
        └── en.lproj/
            ├── index.html              # access page (landing/TOC) — HPDBookAccessPath
            ├── pages/                  # one HTML file per topic
            ├── shrd/
            │   ├── help.css            # Apple-native stylesheet
            │   └── img/                # screenshot placeholders live here
            └── Anglesite.helpindex     # generated by hiutil (gitignored)
```

Help-book `Contents/Info.plist` keys:

| Key | Value |
|---|---|
| `CFBundleIdentifier` | `dev.anglesite.app.help` |
| `CFBundleName` | `Anglesite Help` |
| `CFBundleDevelopmentRegion` | `en` |
| `HPDBookTitle` | `Anglesite Help` |
| `HPDBookType` | `3` (indexed help) |
| `HPDBookAccessPath` | `index.html` |
| `HPDBookIndexPath` | `Anglesite.helpindex` |
| `HPDBookIconPath` | `AnglesiteHelp.png` |

The access path and index path are relative to the `en.lproj` folder.

## Styling (`shrd/help.css`)

Apple-native, not branded (per chosen direction):

- `-apple-system` / system font stack.
- Readable single column, ~640px max width.
- System-style heading scale and link color.
- `@media (prefers-color-scheme: dark)` for light/dark parity with Help Viewer.
- `.screenshot-placeholder` figure style: dashed-border box with a caption, so empty
  screenshot slots read as intentional "image coming" rather than broken images.

## Screenshot placeholders

Screenshots are deferred (chosen direction: placeholders now, capture later). Each
UI-heavy page gets `<figure>` slots containing a `.screenshot-placeholder` box, a real
`<figcaption>`, and the intended `alt` text in an HTML comment so a later capture is a
drop-in. No binary screenshots are committed in this pass. A follow-up task — "capture
Anglesite Help screenshots" — is logged for later.

## Build wiring

1. **`project.yml`** — add a resource folder entry `- path: Resources/Anglesite.help`
   (`type: folder`, resources build phase, `optional: true` to match the existing
   vendored-resource pattern) to **both** the `Anglesite` and `AnglesiteMAS` targets.
2. **`scripts/build-help-index.sh`** — new pre-build script, added to both targets'
   `preBuildScripts`. Runs:
   ```sh
   hiutil -Caf "<lproj>/Anglesite.helpindex" "<lproj>"
   ```
   where `<lproj>` is `Resources/Anglesite.help/Contents/Resources/en.lproj`. Guarded to
   no-op cleanly (warn, exit 0) if `hiutil` is unavailable so the build never breaks on a
   missing tool. Idempotent — writes into the source bundle so the normal resource copy
   picks the index up.
3. **`.gitignore`** — add `Resources/Anglesite.help/**/*.helpindex` (generated artifact,
   like the vendored `node-runtime`).
4. **`Resources/Info.plist`** and **`Resources/AnglesiteMAS-Info.plist`** — add:
   - `CFBundleHelpBookFolder = Anglesite.help`
   - `CFBundleHelpBookName = dev.anglesite.app.help`

   `CFBundleHelpBookName` is the help book's bundle identifier (stable across both the
   DevID and MAS app bundle ids), so a single constant works for both targets.

## Help-menu integration

With the two plist keys present, AppKit registers the book at launch and SwiftUI's
default **Help → Anglesite Help** menu item opens Help Viewer to `index.html`; the
Help-menu **search field** queries the `.helpindex`. **No Swift change is expected.**
During implementation, verify the default item resolves to the book; only if it does not
will we add a `CommandGroup(replacing: .help)` to call `NSApplication.shared.showHelp(_:)`
explicitly. (`AnglesiteApp.swift` already uses `.commands { … }`, so adding a group is
low-risk if needed.)

## Testing / verification

- Build both schemes; confirm the help bundle lands at
  `Anglesite.app/Contents/Resources/Anglesite.help` and the `.helpindex` is generated.
- Launch, open **Help → Anglesite Help**, confirm Help Viewer opens to the landing page
  and intra-book links resolve.
- Type in the Help-menu search field; confirm indexed results appear.
- Check light + dark rendering.
- `scripts/check-help-links.sh` — a lightweight guard that greps every `href`/`src` in
  the help HTML and asserts each local target exists; cheap protection against dead links
  as pages grow.

## Scope guard (YAGNI)

- English only (`en.lproj`); no localization scaffolding.
- No Markdown toolchain — hand-authored HTML.
- No committed screenshots in this pass.
- No online/remote help fallback (`HPDBookKBProduct` / `HPDBookKBURL`).
